# Development Notes

This project is the PayU implementation for building GiveWP/Give payment gateway add-ons.

Use these notes as a checklist when maintaining this gateway or creating another provider-specific gateway.

## Project Shape

- Keep each payment provider in a separate plugin and repository.
- Use a provider-specific plugin slug, text domain, option name, REST namespace and asset names.
- Keep the Give gateway ID short and stable, for example `payu`.
- Include `README.md`, `CHANGELOG.md`, `languages/*.pot`, `languages/*-pl_PL.po` and compiled `.mo`.
- Build release ZIP files with the plugin directory as the top-level folder.
- Keep local release ZIP files in `dist/`, which is ignored by Git.

## Give Integration

- Require the Give plugin with the WordPress `Requires Plugins: give` header.
- Block activation if Give is not active.
- Register Give hooks only when Give is active.
- Add a dedicated settings section under:

```text
Donations > Settings > Payment Gateways
```

- Add a plugin list `Settings` action linking directly to the PayU gateway settings.
- Support Visual Donation Form Builder via the Give payment gateway class.
- Keep the legacy form field markup for compatibility when practical.

## Settings

- Provide sandbox and production modes.
- Store settings in one PayU-specific option array.
- Preserve masked secrets on save.
- Show masked secret fields as `***` after save.
- Add required field markers for required credentials.
- Add a "test API access" action using PayU OAuth `client_credentials`.
- Keep source strings in English and add Polish translations.

## PayU Credentials

- POS ID -> `merchantPosId`
- OAuth client ID -> OAuth `client_id`
- OAuth client secret -> OAuth `client_secret`
- Second key (MD5) -> webhook `OpenPayu-Signature` verification

## Payment Flow

- Create offsite orders through the PayU Orders API.
- Generate a unique `extOrderId` containing the Give donation ID.
- Store the `extOrderId` in donation meta before redirecting the donor.
- Use the Give success page as `continueUrl`.
- Use `/wp-json/give-payu-gateway/v1/status` as `notifyUrl`.
- Use the Give form title in the payment description when available.
- Keep a fallback description using the donation ID.

## Webhook Security

- Verify the PayU `OpenPayu-Signature` header before doing any state changes.
- Parse the donation ID only from the `extOrderId` format generated by the plugin.
- Confirm the webhook `extOrderId` matches the value stored on the donation.
- Confirm amount and currency match the donation.
- Confirm PayU POS ID matches configured credentials.
- Verify the order through the PayU API before completing the donation.
- Confirm verified order status, `extOrderId`, amount, currency and POS ID.
- Make webhook handling idempotent.
- Use a per-donation processing lock to avoid duplicate processing during concurrent callbacks.
- Add stale-lock recovery so a failed process does not permanently block a donation.
- Return `200 ok` for safe duplicate callbacks after a donation is already complete.

## Donation Updates

- Complete the Give donation only after PayU-side order verification succeeds.
- Store useful PayU identifiers in donation meta.
- Add a private donation note with the visible PayU payment ID when available.
- Avoid exposing internal API identifiers in admin notes unless they help support.

## Logging

- Use Give's native log model when available.
- Use category `Payment`.
- Use `PayU` as source.
- Log successful order creation and verification as `success`.
- Log rejected or failed webhook/API states as `error`.
- Log duplicate or non-completed callbacks as `warning`.
- Include structured context, but never log raw secrets.

## Release Checklist

- Run PHP syntax checks.
- Test API access in sandbox.
- Run a full sandbox payment on a public HTTPS WordPress site.
- Confirm status changes from `Pending` to `Complete`.
- Confirm PayU order/payment ID appears in donation meta or note.
- Confirm Give logs have expected type/category/source.
- Build the release ZIP in `dist/`.
- Commit changes.
- Tag the release.
- Create a GitHub Release and upload the ZIP.
- Before final `1.0.0`, run one small live production payment after the PayU account/domain is fully configured.
